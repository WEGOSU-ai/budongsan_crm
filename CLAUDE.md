# 매칭홈 부동산 CRM — 사용 매뉴얼 (CLAUDE.md)

부동산 매물·고객 매칭 관리 도구. **단일 HTML 파일**(`부동산-CRM.html`)로 동작하며 별도 설치·서버 없이 브라우저에서 바로 실행됩니다. 본 문서는 지금까지 반영한 기능과 사용·설정 방법을 한곳에 정리한 매뉴얼입니다.

---

## 1. 빠른 시작

`부동산-CRM.html`을 더블클릭해 브라우저로 엽니다. 처음 화면은 **로그인 게이트**입니다.

- **카카오로 시작하기** — 실제 카카오 로그인. 단, 카카오 개발자센터에 등록한 **https 도메인**에 올렸을 때만 동작합니다(로컬 `file://`에서는 작동하지 않음).
- **로컬·데모로 입장** — 로컬/테스트용. 바로 사용 가능. 평소 PC에서 쓰실 땐 이 버튼을 누르면 됩니다.

로그인 정보(닉네임·이메일)는 헤더 우측에 표시되고, 한 번 입장하면 브라우저에 기억되어 다음 접속 시 게이트를 건너뜁니다. 우측 **로그아웃**으로 초기화됩니다.

> 데이터는 브라우저(localStorage)에 자동 저장됩니다. 같은 PC·같은 브라우저에서는 새로고침/재접속해도 유지됩니다. 다른 PC로 옮기려면 **전체 백업(Excel)** 후 복원하세요.

## 2. 메뉴 구성

상단 탭(모바일은 하단 탭): **대시보드 · 매칭 · 매물 · 고객 · 파이프라인 · 할일**. PC·모바일 반응형이며, 모바일에서는 1열 레이아웃·하단 탭바·우하단 + 버튼으로 전환됩니다.

### 대시보드
- KPI 5장(전체 매물·광고중·전체 고객·계약 진행중·기한 지난 할일). **카드 클릭 시 해당 메뉴로 이동**.
- 일정 캘린더: **월간/주간/일간** 전환. 일간은 06~22시 시간대 표시. 일정에 연락처가 함께 표시되고, 클릭하면 해당 매물·고객·할일 수정 창이 열립니다.
- 임박한 할일: 이름·연락처·텍스트 검색 + 완료 버튼.
- 최근 등록 매물: 매매/전세/월세 필터 + 최신/오래된순 정렬. 클릭 시 매물 수정.
- 데이터 줄: 매물/고객/할일 CSV, **전체 백업(Excel)**, **전체 복원**, **전체 삭제**.

### 매물
- 검색: **모든 단어·모든 필드**(단지·지역·품목·층·동·특징·비고·중개사·용도지역·연락처 숫자까지). 띄어쓴 단어를 **모두 포함**해야 노출(AND 검색).
- 필터: **품목 콤보박스**(보유 품목만 표시), 금액대, 상태.
- 행 클릭 → 수정/삭제. 금액 칸 위에 **거래유형(매매/전세/월세/투자)** 표시.
- 행마다 **📝(블로그 글)** 버튼. 우측 **🗑 매물전체**로 매물만 일괄 삭제(고객·할일 유지). 할일 화면도 **🗑 할일전체** 제공.

### 고객
- 검색: 이름·전화번호(하이픈 무시)·단지(메모). 필터: 거래유형·단계.
- 같은 전화번호는 1회만 표시. 행 클릭 → 수정. 우측 **🗑 고객전체**로 고객만 일괄 삭제.
- 고객 수정창 상단 액션: **💬 안내문구**(단계별 비광고 안내 메시지 생성·복사), **📅 구글 일정**(해당 고객 팔로업 일정으로 구글 캘린더 추가 페이지 열기), **📇 구글 연락처**(vCard 저장 → 구글 주소록 가져오기). **🔚 안내종료**는 고객 목록 각 행의 수정 옆 버튼으로 분리(종료/재개 토글, 종료 시 '종료' 배지).
- 예산 하한·상한은 입력 시 **천 단위 콤마 자동**(예: 55,000).
- **자금 조달·이동 기한** 필드(전량 현금/주담대/디딤돌·버팀목/기존집 처분 등, 3개월 내·6개월 내·즉시·미정)로 진성도를 구조화. 현금+조기이사면 **🔥진성** 배지. 메모·자금·기한에서 주차·재택·학군·반려·역세권·방3+ 등 **니즈 키워드 칩** 자동 추출.
- 저장할 때마다 **수정 히스토리**(변경 항목·시각)가 모달 하단에 누적됩니다.

### 파이프라인
- 신규문의→현장안내→계약검토→계약→사후관리 5단계. PC 드래그, 모바일 이전/다음 버튼. 카드에 연락처 표시, **더블클릭 시 고객 상세로 이동**.

### 할일
- 검색(내용·고객·연락처), 수정/완료, **📅(구글 캘린더 추가)**, 양식 업로드. 날짜는 `2026.06.05` 형식.

## 3. 매물 업로드 — 제목(머리글) 매칭

핵심 원리: **양식이 달라도 "제목(열 머리글)"만 맞으면 그 값을 인식**합니다. 열 순서가 바뀌어도, 일부 칸이 비어도, 한 시트에 여러 섹션(아파트·빌라·상가·건물 등)이 쌓여 있어도 머리글을 자동 감지해 처리합니다.

- 지원: `.csv`, `.xlsx`. 여러 시트도 한 번에 읽습니다.
- 검증 완료 양식: 이지부동산 물건별 양식(11개 섹션), 메인서식 계열(부산 명지·수원 곡반정·강남 건물·목동/화성 상가·명지국제신도시·대연동 등). 수원 곡반정 873건 일괄 인식 확인.
- 업로드 시 **추가/교체** 선택. 섹션 제목·빈 잡행(3자리 이상 숫자가 없는 짧은 행)은 자동 제외.
- 입주일이 "즉시입주"·"2026년 04월 하순"처럼 날짜가 아니면 **비고로 보관**(캘린더 제외).
- 거래구분 "단기임대"→월세, "투자"→투자 유형으로 자동 인식.

### 인식하는 제목(별칭 포함)
매물번호, 부동산종류, 거래구분, 단지명(건물명·매물명), 법정동, 소재지, 지번, 해당층, 평형타입, 동명칭, 전용면적(전용/연면적), 공급면적(공급/대지면적), 매매가격, 보증금, 월세, 방수, 욕실수, 관리비, 난방방식, 세대수(총세대수), 건축년도, 건설회사, 사용승인일, 용도지역, 건축물용도(주용도), 총주차대수, 현재업종, 추천업종, 사용전력, 방향, 입주가능일, **매도자/매수자/임대인/임차인 연락처**, 중개사, 전화번호, 핸드폰, 아이디, 매물특징, 비고.

### 양식 다운로드 / 내보내기
- 매물 화면 **양식** 버튼 → 전체 컬럼 CSV 템플릿. 그대로 채워 올리면 일괄 입력.
- **내보내기** / 대시보드 **전체 백업(Excel)** → 같은 전체 컬럼으로 저장(거래유형에 따라 매매가/보증금 자동 분리).

### 역할별 연락처 (거래유형 자동 전환)
매물 등록창에서 거래구분이 **매매·투자면 매도자·매수자**, **전세·월세면 임대인·임차인** 연락처 칸이 각각 별도로 나타납니다. 목록·매칭·캘린더에는 공급자 측(매도자/임대인) 연락처가 대표로 표시됩니다.

### 품목 종류
아파트·오피스텔·빌라·단독주택·다가구·상가·사무실·건물·공장·창고·지식산업센터·토지·원룸·투룸·전원주택·분양권·재개발/재건축·기타. (코드의 `KINDS` 배열에서 추가 가능)

## 4. 자동 매칭

왼쪽에서 매물(또는 고객)을 선택하면 조건이 맞는 상대를 **0~100점**으로 정렬해 보여줍니다.

- 점수 = 가격 40% + 지역 30% + 평형 20% + 거래유형 10%. 거래유형이 다르면 매칭 제외. **금액(예산 ±10%)이 맞지 않으면 결과에서 제외**해 전달에 방해되지 않게 합니다. 왼쪽 선택 매물 목록은 **중복(매물번호/단지·금액·층 동일)을 1회로** 정리하고, 검색은 단지·지역·품목·매물번호·층·연락처를 함께 훑습니다.
- **월세 환산보증금**: 월세 매칭 시 금액 기준값을 `보증금 + 월세×100`으로 환산해 비교합니다(상가건물 임대차보호법 환산 기준). 결과에 환산보증금이 함께 표시됩니다.
- **평단가**: 아파트·토지 등은 `가격 ÷ 평수(공급 우선, 없으면 전용)`로 평단가(만원/평)를 계산해 매물 기준점으로 보여주고, 매칭 메시지·블로그 글에도 반영합니다.
- 1순위(70점↑)는 **⭐추천**, 85점↑은 **강조색**.
- 같은 연락처/번호는 1회만 표시(중복 제거).
- 왼쪽 필터: 이름·연락처·텍스트·숫자 검색 + **거래유형 칩**(전체/매매/전세/월세/투자) + **품목 칩**(아파트·건물·토지 등 보유 품목).
- 결과 항목 클릭 → 바로 수정. 우측 상단: **✎수정·💬메시지·🖼이미지·⬇CSV**.
- 💬 메시지: 고객이 "번호만 고르면 되도록" 정리된 안내 문구 생성(복사 가능). ⚙에 등록한 중개사무소 이름·대표·연락처가 서명에 자동 포함됩니다.
- 결과마다 **적합도 %**와 고객 **니즈 칩·🔥진성**이 표시되고, **📋 브리핑** 버튼으로 매물+고객 니즈에 맞춘 제안서(초안)를 즉시 생성합니다.
- 대시보드 최근 매물에는 **🎯 추천 고객(이름·적합도%)**이 자동 표기되고, 상단 **💤 재접촉 넛지**로 미계약·미종료 초기 고객 안내 메시지를 일괄 생성합니다. (자동 발송은 도메인+API 연동 시 가능, 현재는 복사 후 발송)

## 5. 데이터 저장·백업·삭제

- **자동 저장**: 모든 변경은 localStorage에 즉시 저장.
- **전체 백업(Excel)**: 매물·고객·할일 3시트 한 파일.
- **전체 복원**: 백업 엑셀을 그대로 되돌림(기존 교체).
- **전체 삭제**: 2중 확인 후 전체 비움(되돌릴 수 없음 — 먼저 백업 권장).
- 개별 CSV 업로드/다운로드도 매물·고객·할일별 제공.

## 6. 개인정보 보호 (마스킹)

헤더 **🔒 가리기**로 전화번호·이름·호수/동을 한 번에 가립니다(예: `***-****-1111`, `김*수`, `래미안 ***동 ****호`). 이 상태로 이미지·PDF 내보내면 가려진 채 저장. 검색은 실제 값으로 계속 동작.

## 7. 중개사무소 정보(⚙) 고정 저장

헤더 **⚙** 버튼에서 **중개사무소 이름·소재지·등록번호·대표 공인중개사·연락처**를 한 번만 등록하면 localStorage에 고정 저장되어 재접속해도 유지됩니다. 등록한 정보는 **블로그 글의 [중개사무소 안내] 블록**과 **매칭 메시지 서명**에 자동으로 채워집니다(등록 전에는 대괄호 자리표시자로 남습니다).

## 8. 블로그 글 생성 (광고법 최소 준수)

매물 목록의 **📝**(또는 수정창 **📝 블로그**) → 해당 매물 정보로 블로그 글 틀 즉시 생성(복사 가능).

「공인중개사법」 제18조의2 및 중개대상물 표시·광고 명시사항 세부기준에 맞춰 **소재지·종류·거래형태·가격·전용면적·층·방향·방/욕실·입주가능일·관리비·사용승인일·주차·건축물용도**와 **중개사무소 명칭·소재지·등록번호·대표 성명·연락처**(⚙ 설정값 자동 반영)를 포함합니다.

> ⚠️ 생성물은 "최소 준수 틀(초안)"입니다. ⚙에서 중개사무소 정보를 채우고, **허위·과장 없이 사실대로** 게시하세요. 위반건축물이면 '위반건축물'을 표시해야 합니다. 게시 전 실제 수치·현장 확인 필요.

근거: 공인중개사법 제18조의2, 국토교통부 「중개대상물의 표시·광고 명시사항 세부기준」.

## 9. 구글 캘린더 추가

할일의 **📅** 버튼(목록·수정창)으로 **구글 캘린더 일정 등록 페이지가 채워진 상태로 새 탭에 열립니다**(클릭 후 저장하면 구글 알림으로 동기화). 제목·날짜·고객·연락처·시간이 자동 입력됩니다.

> 카카오톡 일정은 외부에서 직접 등록할 수 있는 공개 링크가 없어 지원하지 않습니다. 구글 캘린더 일정은 구글 앱 알림으로 받을 수 있습니다.

## 9.5 통화녹음 자동 분석(STT) 시스템

휴대폰 통화녹음이 자동으로 고객 카드가 되는 파이프라인입니다.

**구성.** 로컬 변환서버(`budongsan_stt_server.py`, 127.0.0.1:8787, faster-whisper GPU) + 통화워커(`run_worker.py`, OneDrive 통화녹음 폴더 20초 감시) + ollama 분석모델(현재 `batiai/gemma4-12b:iq3`). 흐름: 폰 OneDrive 자동업로드 → 워커 감지 → whisper 변환 → AI 분석 → **매칭홈 클라우드 고객명단 자동 등록**. 매칭홈 통화업로드 탭의 수동 드래그 업로드도 같은 서버로 동일하게 분석됩니다.

**카드에 채워지는 것.** 이름(`이름_연락처_통화일시`), 거래유형·입장·지역·예산·월세·평형·자금·이동기한·입주희망일, **분류(통화 자동)** 5종(스팸/오접속 · 실수요-계약 진행중 · 실수요-매물문의 · 임대인/공동중개 · 타업종 영업/비매물 — 고객 화면 "분류" 필터), **실행 액션(요약)**에 통화별 `[일시] 액션 · 주의` 누적, 메모에 통화 블록(`[일시 통화 #n] → 녹음파일명 → 요약 → 핵심발언(결정 발언 원문 3~5개) → 전문(화자분리)`)이 **통화 시각순**으로 쌓입니다. 같은 번호는 한 카드로 병합(전화+통화시각 중복 방지), 고객 목록 기본 정렬은 **최근통화순**(🎤 배지에 마지막 통화 시각 표시).

**범위·예외.** 2026-01-01 이전 옛 녹음은 건너뜀(`run_worker.py`의 MIN_FILE_DATE로 조정). 30분 넘는 긴 통화는 자동 구간 분할로 요약·화자분리(유실 없음). 부동산 무관 통화도 `[일반통화]` 요약으로 등록.

**운영.** 부팅 자동 시작(시작프로그램 `matchinghome-autostart.cmd` → server-start.bat + worker-start.bat 최소화). 수동 재시작 = `_kill.bat` → `server-start.bat` → `worker-start.bat`. 모델 변경 = `%LOCALAPPDATA%\matchinghome\stt_model.txt`에 ollama 모델명 기록 후 재시작(`_model-cleanup.bat`=옛 모델 정리+현재 모델 고정, `_model-eeve.bat`=eeve 복귀). 기존 카드 일괄 재분류 = `_reclassify.bat`(미분류만, 재실행 안전).

**주의.** 매칭홈은 저장 시 클라우드 전체 교체 방식 — **매칭홈 작업 시작 전 F5(새로고침)** 습관이 그 사이 자동 등록된 통화를 지킵니다.

## 11. 제약·주의

- **저장 위치**: 게스트는 브라우저 localStorage, 클라우드 로그인 시 Supabase 동기화(권장). 엑셀 일괄 백업 버튼은 제거됨(클라우드가 대체) — 필요 시 매물/고객/할일 CSV 내보내기 사용.
- **인터넷 필요 기능**: 엑셀 업로드/백업, 이미지 복사, PDF 저장, 카카오 로그인은 외부 라이브러리·SDK 로드 필요(오프라인에선 CSV 업/다운로드 가능).
- **카카오 로그인**: https 도메인 등록 필수. 이메일·전화번호 수집은 비즈앱 전환 + 동의항목 **검수** 통과 필요(닉네임 등 기본 프로필은 검수 없이 가능).
- **구글 캘린더**: 일정 등록 페이지를 여는 방식(원클릭 후 저장). 서버 없이 동작.
- **국토교통부 전국 법정동 CSV**: 용량이 커서 내장하지 않았습니다. 법정동 자동완성·표준화가 필요하면 경량 사전(시·군·구 단위)으로 별도 연결 권장.

## 12. 확장 방법

- **다른 양식 추가**: 코드의 `PKEYS`(매물)·`mapCust`/`mapTask` 별칭에 양식의 제목 키워드 추가.
- **품목 추가**: `KINDS` 배열에 항목 추가 → 모달·필터·매칭 칩에 함께 반영.
- **거래유형**: 현재 매매·전세·월세·투자(+단기임대→월세). `TYPE`·`toType` 확장.
